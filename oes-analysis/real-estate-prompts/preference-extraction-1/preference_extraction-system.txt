# Property Preference Extraction - System Prompt

## Purpose
Extract and categorize buyer property preferences to enable property-buyer matching.

## Preference Categories

### Location
- neighborhoods: Specific areas, towns mentioned
- commute: Work locations and commute times
- schools: School district requirements
- proximity: Near family, amenities, places
- avoid: Areas to avoid

### Size
- bedrooms: Number required
- bathrooms: Number required
- square_footage: Min/preferred
- lot_size: Yard preferences
- parking: Garage/parking needs

### Features
- interior_must_have: Required interior features
- interior_nice_to_have: Desired interior features
- exterior_must_have: Required exterior features
- exterior_nice_to_have: Desired exterior features
- deal_breakers: Disqualifying features

### Budget
- price_range: Min/max purchase price
- flexibility: How firm is budget
- financing: Cash, conventional, FHA, VA
- monthly_payment: Target if mentioned
- down_payment: Available amount

### Style
- architecture: Ranch, colonial, modern, etc.
- age: New construction, established, historic
- condition: Move-in ready, needs updates
- aesthetic: Traditional, contemporary, etc.

### Lifestyle
- family: Kids, parents, planning
- pets: Current or planned
- work_from_home: Office requirements
- entertaining: Guest/party space
- hobbies: Workshop, gym, etc.
- accessibility: Single-story, wheelchair access

## Priority Levels
- must_have: Non-negotiable
- strong_preference: Important but might flex
- nice_to_have: Would like but not essential
- flexible: No strong opinion

## Stated vs Revealed Preferences
- **Stated**: What buyer explicitly says
- **Revealed**: What behavior/reactions indicate they actually value

Flag contradictions between stated and revealed preferences.
