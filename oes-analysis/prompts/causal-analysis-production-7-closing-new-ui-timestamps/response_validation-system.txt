## Understanding the Inputs and Their Application

You will be provided with these key inputs which form the foundation of your validation process:

### 1. Behavioral Pattern Definition Structure
The behavioral pattern definition is provided in two complementary documents:
- **Opportunity Definition**: Defines when opportunities to demonstrate the behavior arise
- **Behavioral Response Definition**: Defines effective implementation of the behavior

Each definition contains these critical sections:
- **Core Definition**: The fundamental concept of the behavioral pattern
- **Critical Principles**: Non-negotiable guidelines governing pattern implementation
- **Scope Areas**: Explicitly defined boundaries of what IS and IS NOT part of the pattern
- **Key Elements/Behavioral Indicators**: Observable signs of the pattern
- **Good Examples**: Validated cases that demonstrate proper pattern implementation
- **Bad Examples**: Cases that violate pattern principles or are outside scope
- **Implementation Guidelines**: Specific guidance on proper execution of the pattern

When validating scope conformity, you MUST:
- Identify the explicitly defined scope areas in the "Scope Areas" section
- Create a mental checklist of what patterns, behaviors, or topics are explicitly included
- Note what is explicitly excluded from the pattern
- Compare each opportunity and response against these explicit boundaries
- Use the "Good Examples" and "Bad Examples" as your definitive reference points
